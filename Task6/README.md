### **Метрики для оценки эффективности**

| **№** | **Метрика**                              | **Цель использования для оптимизации**                                                                    |
|:-----:|:-----------------------------------------|:----------------------------------------------------------------------------------------------------------|
|   1   | Доля корректно классифицированных данных | Оценка и доработка правил классификации, определение ответственных групп за конкретные типы тегов         |
|   2   | Среднее время простановки одного тега    | Выявление узких мест и упрощение процессов тегирования, обоснование потребности в дополнительных ресурсах |
|   3   | Частота использования различных тегов    | Анализ востребованности тегов для принятия решения о введении новых или упразднении неиспользуемых        |
|   4   | Количество ошибок в процессе тегирования | Диагностика корневых причин проблем (недостаточность обучения, сложные правила, технические сбои)         |

### **Обеспечение scalability системы**

Для обеспечения масштабируемости процесса классификации данных предлагается разделить его на логические этапы:

* **Предварительная категоризация:** Определение группы данных на основе атрибутов (например, тип контента: изображение,
  текст, таблица).
* **Классификация по конфиденциальности:** Присвоение тегов, определяющих уровень чувствительности информации (PII, PHI,
  коммерческая тайна).
* **Административная маркировка:** Простановка служебных тегов для управления жизненным циклом (например, для
  архивирования, определения срока хранения).

Каждому этапу может соответствовать отдельный сервис или модуль. Это позволяет независимо масштабировать ресурсы для
наиболее загруженного этапа. Для повышения производительности рекомендуется:

* Внедрить кеширование правил классификации (например, с использованием Redis) для снижения нагрузки на основные
  системы.
* Организовать отдельное хранилище для исторических и архивных данных. Часть данных (например, при массовой миграции из
  legacy-систем) можно направлять туда напрямую, минуя основные обработчики.
