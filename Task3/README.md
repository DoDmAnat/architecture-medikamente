### **Cписок данных для защиты**

| **№** | **Данные**               | **При хранении**           | **Во время передачи** | **В чём необходимость защиты**                                                                                                                                                                                                                                        |
|:-----:|:-------------------------|----------------------------|-----------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|   1   | Данные пациентов         | Шифрование на уровне диска | SSL/TLS               | Информация должна быть доступна пациенту через личный кабинет, что подразумевает внешний доступ. Необходимо шифрование при передаче для защиты от перехвата, а также при хранении — чтобы похищенные файлы нельзя было прочесть.                                      |
|   2   | Журналы приёма пациентов |                            | SSL/TLS               | Применение обфускации идентификатора карты в записях делает их бесполезными без доступа к основным данным пациентов. Поэтому шифрование при хранении не является приоритетным, однако при передаче оно необходимо для предотвращения использования украденных данных. |
|   3   | Медицинские карты        | Шифрование на уровне диска | SSL/TLS, VPN          | Данные должны быть доступны пациенту через личный кабинет, что означает их передачу по внешним каналам. Требуется шифрование при передаче и хранении, чтобы в случае кражи файлы оставались нечитаемыми.                                                              |
|   4   | Результаты анализов      |                            | SSL/TLS               | Поскольку в анализах используется обфускация идентификатора карты, шифрование на уровне хранения не считается обязательным.                                                                                                                                           |

платформы **OneTrust**.
